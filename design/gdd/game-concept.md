## 1. 项目身份 (Project Identity)

| 项目 | 内容 |
|------|------|
| **项目名称** | 《糖大冒险：雪山狂飙》(Sugar Adventure: Snow Mountain Rush) |
| **品类** | 2D 滑雪跑酷 (2D Side-scrolling Skiing Runner) |
| **题材** | Serious Game / 健康教育游戏 |
| **主题** | 糖尿病健康管理（血糖、饮食、运动、用药） |
| **核心玩法参考** | 《滑雪大冒险》(Ski Safari) |
| **项目性质** | 课程项目（具备产品化扩展潜力） |
| **团队规模** | 2-3 人 |
| **时间预算** | < 1 个月（Game Jam 节奏 / 垂直切片导向） |
| **目标引擎** | Unity 2022.3 LTS（2D Core 模板，已建于 unity-project/） |
| **目标平台** | PC (Windows 主 / Mac+Linux 可选) |
| **审查模式** | Lean（精简审查，仅在阶段门槛进行 Director 审查） |

**一句话定位**
> 一款把血糖管理变成游戏机制的 2D 横版滑雪跑酷游戏 —— 玩家在滑、跳、翻滚、躲、收集的过程中，自然理解糖尿病控糖的关键概念。

## 2. 核心愿景 (Core Vision)

设计哲学概括为三个关键词的平衡三角：

```
              幽默 (Humor)
                  ▲
                /   \
               /     \
              /       \
   爽感 (Feel) ——————— 教育 (Education)
```

| 关键词 | 实现方式 | 防止滑向 |
|--------|---------|---------|
| **幽默** | 卡通角色、夸张医疗道具、搞笑语音、奇幻医疗雪山 | 化解题材沉重感，但不要轻佻到伤害医学话题尊严 |
| **教育** | 每个游戏机制对应真实糖尿病知识（机制转译） | 避免说教化、避免变成"知识贴牌的游戏" |
| **爽感** | 跑酷反馈感：流畅滑行、跳跃、翻滚、躲避、收集 | 教育不打断节奏，知识嵌入而非弹窗 |

**核心信条**：寓教于乐 — "玩着玩着懂了控糖"。

> ⚠️ **待团队讨论确认**。以下为助理建议的两个主选方向，建议二选一以聚焦设计：

### 选项 A：青少年 2 型糖尿病科普

## 4. 玩家幻想 (Player Fantasy)

**玩家进入游戏时应该感觉**：
- 我是一个掌握命运的小英雄（"糖糖"），在医疗雪山高速滑行
- 周围环境既奇幻又危险 —— 红色风雪代表危机、冰蓝冰窟让我谨慎、明亮营地给我回血
- 我要在飞速躲避中做选择：吃这片高糖雪花换速度吗？现在用胰岛素喷雾还是再等等？
- 通关时感觉：我赢了一局快节奏滑雪游戏，**同时**我下意识理解了"血糖平衡 = 生存"这件事

**避免的玩家感受**：
- ❌ "这是一堂披着游戏外皮的健康课"
- ❌ "我在被教育"
- ❌ "节奏被科普弹窗打断"

## 5. 核心循环 (Core Loop)

```
┌─────────────────────────────────────────────┐
│  滑行 (Slide)                              │
│   │                                         │
│   ▼                                         │
│  实时血糖变化 (Glucose Fluctuation)         │
│   │                                         │
│   ├── 玩家行为（吃高糖雪花 ↑ / 运动靴 ↓）  │
│   ├── 道具效果（胰岛素 / 降糖药 ↓）         │
│   └── 区域环境（高血糖区被动 ↑）            │
│   │                                         │
│   ▼                                         │
│  决策 (Decision)                            │
│   │                                         │
│   ├── 跳跃 / 翻滚 / 转向                   │
│   ├── 用道具 / 存道具                       │
│   └── 走稳路 / 走捷径但风险高               │
│   │                                         │
│   ▼                                         │
│  反馈 (Feedback)                            │
│   │                                         │
│   ├── 视觉：血糖条颜色变化                  │
│   ├── 听觉：心跳音 / 警报音                 │
│   └── 物理：滑行速度 / 操控感变化           │
│   │                                         │
│   ▼                                         │
│  抵达「医院降糖站」(Goal Reached) → 评分    │
└─────────────────────────────────────────────┘
              │
              ▼ (重玩/下一关)
```

**循环周期**：单关 60-120 秒（适合移动端碎片时间）

## 6. 独特卖点 (USP: Unique Selling Proposition)

**机制转译 (Mechanic Translation)** — 不是"知识贴在游戏外面"，而是"医学知识就是游戏变量本身"。

| 对比 | 普通健康教育游戏 | 本项目 |
|------|----------------|--------|
| 知识呈现 | 关卡间弹窗、教学卡片 | 玩家必须实时处理的游戏变量 |
| 学习方式 | 阅读 / 选择题 | 操作中自然理解因果 |
| 关卡设计 | 通用关卡 + 知识题 | 关卡本身就是医学概念的视觉化（高血糖区 = 红色狂暴风雪） |
| 道具设计 | 通用 buff/debuff | 每个道具都是医学概念（胰岛素 = 救命符、高糖食物 = 双刃剑） |

这种"机制 = 内容"的设计是 Serious Game 评估时的最高得分项。

## 7. 机制转译矩阵 🌟 (Mechanic Translation Matrix)

**这是本项目最大的设计资产，所有后续系统设计都应回到这张表对齐**：

| # | 医学概念 | 游戏化身 | 玩家感受 | 学习目标 |
|---|---------|---------|---------|---------|
| 1 | 血糖水平 | 实时状态条（核心机制） | 必须时时刻刻关注的"生存条" | 血糖平衡的重要性 |
| 2 | 高糖饮食 | 高糖雪花（双刃剑道具） | "吃了爽但有代价"的选择压力 | 高糖食物的风险 |
| 3 | 胰岛素治疗 | 胰岛素喷雾（救命道具） | 危急时刻的安全网 | 胰岛素的作用与时机 |
| 4 | 口服降糖药 | 降糖药瓶（持续型道具） | 长效防控 vs 短期救急的区别 | 不同药物的作用差异 |
| 5 | 运动干预 | 运动靴（机动性增益） | 运动让我更敏捷 | 运动对控糖的正面作用 |
| 6 | 自我监测 | 血糖计（信息 + 冲刺奖励） | 监测让我心里有数 | 定期监测的价值 |
| 7 | 高血糖危机 | 红色狂暴雪区（环境压力） | 视觉化的"血糖飙高" | 高血糖的危险与症状 |
| 8 | 低血糖危机 | 冰蓝冰窟（操控阻力） | 视觉化的"低血糖虚弱" | 低血糖的危险与症状 |
| 9 | 健康干预 | 减肥营地（绿色 / 温暖） | 干预成功的正反馈 | 生活方式改变的积极意义 |

## 8. 世界观与场景 (World & Setting)

**世界设定**：医疗雪山 — 一个由医疗元素构成的奇幻雪山世界。
不同区域象征糖尿病管理中的不同状态和风险。

### 三大核心区域

| 区域 | 视觉关键词 | 玩法关键词 | 象征意义 |
|------|----------|----------|---------|
| **高血糖狂暴雪区** | 红色、暴风雪、棱角障碍密集 | 速度更快、障碍多 | 高血糖带来的失控感 |
| **低血糖冰窟** | 冰蓝、雾化、模糊视野 | 滑行阻力大、视线短 | 低血糖的虚弱迟缓 |
| **减肥营地** | 明亮、温暖、绿色植物、运动器材 | 跳跃距离奖励、安全区 | 运动干预的正反馈 |

### 终点：医院降糖站
- 每关都需要抵达的"安全港"
- 视觉风格统一：白色 + 绿色十字 + 友好医院造型

## 9. 关卡规划 (Level Plan)

### 原案完整规划（5-8 关）

| 关卡 | 主题 | 核心目标 |
|------|------|---------|
| L1 | 医院雪屋（入门关） | 熟悉基础操作 |
| L2 | 高血糖狂暴雪区初体验 | 学会用胰岛素喷雾 |
| L3 | 低血糖冰窟初体验 | 学会用高糖雪花救低血糖 |
| L4 | 综合区域 | 双向血糖控制 |
| L5 | 减肥营地 | 运动靴的应用 |
| L6+ | 进阶 / 挑战关卡 | 多机制综合 |

### ⭐ MVP 范围（< 1 个月 jam 计划）

**只做 1-3 关**，按优先级：

1. **L1 医院雪屋（必做）** — 验证核心循环
2. **L2 高血糖区（强烈推荐）** — 展示机制转译爽点
3. **L3 低血糖区（如时间允许）** — 完整双向血糖体验

**砍掉**：L4 / L5 / L6+ 进入"扩展规划"。

## 10. 主角设计 (Character: 糖糖) + 反刻板说明

### 基本设定
- **名字**：糖糖（Tangtang）
- **角色定位**：玩家代入的中性卡通形象

### ⚠️ 反刻板设计原则（针对策划案识别的风险 3）

**避免**：把糖糖设计为"小胖子"以强化"肥胖 = 糖尿病"的简单理解。

**建议方案**（团队三选一）：

#### 方案 A：中性卡通形象（最安全）
- 圆滚 mascot 风格，不强调真实人体比例
- 像 Among Us 那样去人化的萌系造型

#### 方案 B：多角色可选
- 提供 3-4 个不同体型、年龄、性别的可选角色
- 传达"糖尿病不分人群"

#### 方案 C：拟物化主角
- 把主角设计成一个"小药丸"或"小血糖滴"
- 完全跳过人体表达，去刻板更彻底

> 团队需在 W1 决定方案，影响美术资产采购方向。

## 11. 道具系统 (Item System)

### 完整 5 道具表

| 道具 | 现实映射 | 游戏效果 | MVP 优先级 |
|------|---------|---------|----------|
| **胰岛素喷雾** | 紧急胰岛素 | 立即降低血糖 + 修复警戒状态 | ⭐ MVP 必做 |
| **降糖药瓶** | 口服降糖药 | 持续降低血糖（持续 X 秒） | ⭐ MVP 必做 |
| **血糖计** | 监测设备 | 显示精确血糖值 + 短暂冲刺 | 🔵 W3 加入 |
| **高糖雪花** | 高糖食物 | 短暂加速但提高血糖（双刃剑） | ⭐ MVP 必做 |
| **运动靴** | 运动干预 | 增加跳跃距离 + 滞空时间 | 🔵 W3 加入 |

**MVP 必做 3 个**：胰岛素喷雾 + 降糖药瓶 + 高糖雪花
（这 3 个构成完整的"血糖三角"：升高源、急救、慢救）

## 12. 操作设计 (Controls)

| 操作 | 键鼠 | 手柄 (Xbox/PS 映射) |
|------|------|---------------------|
| 跳跃 | Space / 鼠标左键 | A 键 / × |
| 翻滚 | Shift / 鼠标右键 | B 键 / ○ |
| 调整方向 | A/D 键 或 ← / → | 左摇杆 / 方向键 |
| 使用道具 | Q / E / 数字键 1-3 | LB / RB / Y / △ |
| 暂停 | Esc | Start / Options |

**设计原则**：
- 键盘单手 + 鼠标可操作（左手 WASD + Space + Shift，右手鼠标）
- 完整手柄支持（Xbox/PS 标准映射）
- 学习曲线极低（≤ 30 秒理解所有操作）
- 决策窗口短但可读（玩家有 0.5-1 秒看到障碍并反应）
- 所有 UI 必须支持键盘导航（无 hover-only 交互）

## 13. 单人 / 多人模式

### 单人模式 ⭐ MVP 核心
- **目标**：限时抵达终点 + 血糖不失控
- **完美通关条件**：
  - 血糖始终保持正常（不进入警戒状态）
  - 不摔倒
  - 完成 N 个翻滚特技
- **评级**：S / A / B / C 四档

### 多人模式 ❌ MVP 不做（移入扩展规划）
原案设计：
- 2-4 人局域网
- 竞速 + 高糖炸弹干扰
- 合作模式：投掷急救包 / 降糖药球互救

**砍掉理由**：
- LAN 同步 + 状态同步 + 干扰效果同步对 2-3 人 / < 1 月 团队风险过高
- 单人模式已能完整展示"机制转译"设计亮点
- 课程答辩看的是"机制是否成立"，不是"模式是否完整"

## 14. 教育目标与 KPI (Educational Goals & KPIs)

> ⚠️ **策划案识别的风险 4**：教育效果验证不够明确。以下为助理草案，团队需在 W1 确认。

### 可测量的教育目标
玩家通关 MVP 后应能正确回答以下问题（建议作为试玩问卷）：

1. **问**：当血糖过高时，游戏中应该使用哪个道具？
   **答**：胰岛素喷雾（对应：高血糖时用胰岛素降糖）

2. **问**：高糖雪花为什么是"双刃剑"？
   **答**：短期加速但会提高血糖风险（对应：高糖食物风险）

3. **问**：游戏中的"运动靴"代表生活中的什么？
   **答**：运动干预对血糖控制的积极作用

4. **问**："血糖计"道具有什么作用？
   **答**：监测血糖（对应：糖尿病患者需要定期监测）

5. **问**：为什么红色狂暴雪区障碍更多？
   **答**：象征高血糖带来的危机和并发症风险

### 量化 KPI（建议）
- 试玩玩家中 ≥ 70% 能正确回答上述 5 题中的 4 题
- 试玩玩家中 ≥ 80% 反馈"游戏帮我理解了血糖管理"
- 课程答辩演示中能清晰说明每个机制的医学映射

## 15. 医学准确性策略 (Medical Accuracy Strategy)

**团队决策**：课程标准 — 游戏化合理即可，不追求临床级准确（用户在引导阶段已选 C）。

### 操作原则
1. **避免误导性的具体数值**：不要在 UI 显示具体的 mmol/L 血糖值，用"低/正常/偏高/高/危险"5 档分级
2. **避免错误的剂量暗示**：胰岛素喷雾的"一次使用降低 X 单位"不要对应真实剂量
3. **保留概念正确性**：高血糖症状、低血糖症状、运动有益、监测重要 — 这些大方向不能错
4. **关键文案审核**：道具说明、关卡介绍、教程提示 — 团队内部交叉审核，确保没有明显医学错误

### 课程标准 ≠ 随意编造
- 参考资料：中国糖尿病防治指南摘要 / 公开科普文章
- 避免：胰岛素和口服药的混淆、低血糖和高血糖症状的混淆、运动 = 万能等过度简化

## 18. ⭐ MVP 范围 (4 周计划)

### 周计划

| 周 | 阶段 | 关键产出 | 风险/检查点 |
|----|------|---------|-----------|
| **W1** | 引擎 + 原型 | Unity 工程搭建、滑行控制器、血糖状态条原型 | 控制器手感是否爽？血糖逻辑是否成立？ |
| **W2** | 核心机制 | 血糖系统完整、3 个核心道具、L1 医院雪屋可玩 | 第一次内部试玩 — 验证机制转译易懂吗？ |
| **W3** | 内容扩展 | L2 高血糖区、UI 完整化、音效集成 | L3 视情况加入 |
| **W4** | 打磨 + 演示 | Bug 修复、平衡调整、Demo 录制、答辩材料 | 课程答辩准备 |

### MVP 必含内容清单

| 类别 | MVP 必含 | 不含（扩展） |
|------|---------|-----------|
| 模式 | 单人 | 多人、合作 |
| 关卡 | L1 必、L2 强推、L3 弹性 | L4-L8 |
| 道具 | 胰岛素喷雾、降糖药瓶、高糖雪花 | 血糖计、运动靴（W3 弹性） |
| 区域视觉 | 1-2 个 | 完整 3 区 |
| UI | 血糖条、距离、暂停、结算 | 商店、收集图鉴、成就 |
| 音效 | 跳跃、收集、警戒、失败 | 配音、角色台词 |
| 教程 | 关内提示 | 独立新手引导关 |
| 评分 | 完美通关条件 | 在线排行榜 |

## 19. 扩展规划 (Out-of-MVP Extensions)

按优先级分组的"做完 MVP 还有时间就考虑"：

### 🟢 P1 — 容易加，价值高
- 新手引导模块
- 控糖成就系统
- 医疗百科图鉴收集
- 第 3 关「减肥营地」

### 🟡 P2 — 中等复杂度
- 角色多选（不同体型 / 年龄 / 性别）
- 关卡进阶版（饮食迷宫、运动公园、医院大冒险）
- 新角色：医生、护士 NPC
- 难度调节（简单 / 普通 / 困难）

### 🔴 P3 — 大工程
- **多人模式**（2-4 人局域网）
- 在线排行榜
- 每日挑战、好友对战
- 与医疗机构正式合作
- 多语言、多平台移植

## 20. 已识别风险与缓解策略 (Risks & Mitigation)

| # | 风险 | 严重度 | 缓解策略 |
|---|------|-------|---------|
| 1 | **医学准确性** — 可能误导玩家健康决策 | 🔴 高 | 课程标准 + 关键文案内部审核 + 避免具体数值 + 参考中国糖尿病防治指南 |
| 2 | **多人模式开发复杂度** — 拖慢 MVP | 🟡 中 | ✅ 已决策：多人移入扩展规划 |
| 3 | **角色刻板化** — "胖 = 糖尿病" 强化偏见 | 🟡 中 | 糖糖采用方案 A/B/C 三选一（W1 决定）+ 文案避免体型描述 |
| 4 | **教育效果验证缺失** — 学没学到东西无法判断 | 🟡 中 | 设定 5 道试玩问卷 + 课程答辩中演示机制转译 |
| 5 | **范围超出时间预算** — < 1 月做不完 | 🔴 高 | MVP 砍至 1-3 关 + 3 道具 + 1-2 视觉区 + 优先 Asset Store 采购 |
| 6 | **目标人群未定** — 影响 UI/难度/文案 | 🟡 中 | W1 第一次团队会议必须决定（建议成人大众科普） |

## 21. 明确超出范围 (Out of Scope for MVP)

为避免范围蔓延 (scope creep)，以下内容明确**不做**：

- ❌ 多人模式（所有联网功能）
- ❌ 程序化地形生成（改为手作关卡序列）
- ❌ 角色商店 / 内购系统
- ❌ 在线功能（排行榜、好友、云存档）
- ❌ 数据收集与分析后台
- ❌ 多语言支持（中文优先）
- ❌ 多平台同步发布（先做单一平台）
- ❌ 复杂剧情过场 / 配音
- ❌ 与真实医疗机构的合作签约（产品化阶段再考虑）
