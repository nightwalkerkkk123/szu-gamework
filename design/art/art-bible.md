## Section 1: Visual Identity Statement

### 视觉决策总则

> **「卡通奇幻游乐场 — 不让医学写实侵入,也不走 Alto 式留白极简;圆润 Q 版、鲜艳三色、夸张动效优先」**

当任何视觉决策出现歧义时,这一句就是裁决。

---

### 支撑原则

#### 原则 1 — 角色与表情(锚定:幽默 + 教育)

> 糖糖和 NPC 圆润 Q 版、表情夸张搞笑,拒绝写实医疗刻板印象。

**设计测试**:美术纠结"这个道具要画成写实针筒还是卡通药瓶"时,选卡通版本。注射器可以变成一个会尖叫的蓝色小怪物,但绝不能是医院里冷冰冰的真实注射器。

#### 原则 2 — 三区配色锁定情绪语义(锚定:教育 + 爽感)

> 红区 = 危险但有趣、蓝区 = 冷静但可逆、绿区 = 安全且有奖励 — 颜色即是游戏状态语言。

**设计测试**:新道具或 UI 元素配色与所在区域冲突时,先服从区域主色调。绿区里一个红色道具必须附带强解释成本(闪烁/特效),否则改用绿/黄色系。

#### 原则 3 — UI 显眼且即时反馈(锚定:教育 + 爽感)

> 血糖条始终可见且面积够大;道具效果伴随夸张动效;数值变化必有动画,绝无静默。

**设计测试**:UI 和游戏画面争夺注意力时,UI 必须赢 — 血糖条可以在画面边缘但不能被遮挡,收集道具的 +/− 数值必须弹出放大 0.3 秒。

---

### Rationale

这套规则同时服务于三支柱:

- **幽默** → Q 版角色 + 夸张道具 + 鲜艳卡通拒绝医疗题材的沉重感。糖糖是个"会翻跟头的快乐小糖人",不是"病房里的患者"。
- **教育** → 三区配色作为血糖状态的视觉隐喻,玩家在区域切换中自然感受"红区危险→蓝区危险→绿区安全"的医学逻辑,不需读文字。
- **爽感** → 圆润 sprite 在 parallax 雪山中滑行、跳跃、翻滚,视觉节奏与跑酷节奏共振。

**关于糖尿病题材的尊严**:规则刻意强调"卡通奇幻游乐场"而非"游戏化诊所" — 这确保视觉语言尊重受众,不让玩家感觉被教育或被污名化。Q 版糖糖不是"病人",是一个在雪山冒险的普通小孩。

**关于 jam 节奏可执行性**:规则足够宽松以容纳 Asset Store 采购资产(卡通风格 2D 素材高度匹配),同时足够具体以排除不匹配选项(写实医疗、超简约 Alto 风),减少团队决策摩擦。

### 视觉连续性原则

为确保状态间过渡自然且可读,以下通则必须遵守:

**状态切换插值:**

| 过渡方向 | 插值时长 | 天空渐变速度 | 粒子过渡方式 |
|----------|----------|--------------|--------------|
| 正常 → 警戒 | 0.8s | 快速混合 | 原有粒子 + 警示粒子叠加(不突然替换) |
| 警戒 → 危机 | 0.5s | 快速加深 | 警示粒子频率加快 → 替换为风暴粒子 + **漫画速度线进入** |
| 危机 → 失败 | 1.2s | 缓慢坍缩 | 风暴渐停 → 灰烬/糖晶碎片承接 |
| 任意 → 道具收集 | 0.3s | 无变化 | 即时爆发(覆盖原粒子 0.5s) |
| 任意 → 胜利 | 1.0s | 渐变为金橙 | 原粒子淡出 → 礼花粒子承接 |

**三区域核心差异速查:**

| 区域 | 主色调 | 天空对比度 | 粒子特征 | 后期风格 |
|------|--------|------------|----------|----------|
| L1 医院雪屋 | 蓝绿白 | 弱 | 细雪 + 草药蒸汽 | 干净、冷调 |
| L2 高血糖狂暴 | 红橙黑 | 极强 | 红色暴风雪 + 闪电 | 躁动、饱和、抖动 |
| L3 低血糖冰窟 | 深蓝白 | 弱 | 浓雾 + 大颗粒冰晶 | 模糊、黏稠、沉静 |

### 题材尊严守则(本节专属)

- **失败状态绝不模拟真实低/高血糖症状**(无视觉模糊、无心电图、无去饱和到死灰)
- **危机状态用漫画语言表达紧迫感**(分镜框、速度线、❗❗❗、螺旋眼)
- **L3 冰窟的"视野模糊"是 卡通积雪/雾化效果,不是 真实眼前发黑**
- 所有警戒/危机/失败文案:**用卡通气泡字体 + 轻松语气**,绝不出现"血糖危险"、"立即送医"等真实医疗用语

## Section 3: Shape Language

几何视觉词汇 —— 定义本游戏的形状语言,从角色剪影到环境到 UI 到主辅关系。

### 子主题 1: 角色剪影哲学 (Character Silhouette Philosophy)

**核心主张**:**"1cm 缩略图测试":每个可操控/可交互角色必须在 1cm × 1cm 下 5 秒内被辨认。** 剪影是身份的指纹,不是装饰。

#### 糖糖 (主角) — 辨识特征

| 特征 | 设计规格 | 辨识原理 |
|------|----------|----------|
| **轮廓** | 汤圆形 (soup dumpling) — 宽高比约 1:1.1,顶视图微扁 | 圆润 = 无威胁 = 亲近感,Q 版基准形 |
| **头顶** | 单枚 oversized 红色毛线帽(尖顶+毛球) | 帽形在远景滑雪视角下形成最强轮廓钉 |
| **四肢** | 简化胶囊臂(无手指)+ 雪杖圆角矩形 | 2D 横版内轮廓简化到极致仍保留"人型感" |
| **配色** | 白底 + 红帽 + 深棕靴 | 三色原则在剪影层面的投影 |

**头身比建议:2.5 头身**
- 2 头身:过于卡通,可能与学龄前 IP 混淆
- 3 头身:更像"正常人类低幼版",不符合糖糖作为血糖化身的独特感
- **2.5 头身** = 医疗中性吉祥物的最优解

**缩略图测试通过标准**:糖糖倒伏在雪地时,红帽毛球 + 白圆身仍能在小图标中辨认为"滑雪小人",而非"雪球"或"棉花糖"。

#### NPC (沿途角色) — 辨识矩阵

| NPC 类型 | 主导轮廓 | 区分特征 | 情绪信号 |
|--------|----------|----------|----------|
| **胰岛素精灵** | 泪滴形 (teardrop) | 蓝色水滴袍 + 小翅膀 | 友善/帮助 |
| **血糖仪精灵** | 圆角矩形(机器感) | 屏幕 + 按键 + 红色点缀 | 中性/指示 |
| **饮食辅导员** | 梨形 (pear) | 绿色运动服 + 秒表 | 活力/催促 |
| **Boss:血糖巨魔** | 不规则多边形 + 尖角 | 红色雾气缠绕 + 黑色轮廓线 | 威胁/对抗 |

**设计原则**:NPC 使用**单一几何原型 + 差异化配件**,而非全异轮廓。横版跑酷中玩家注意力在糖糖和障碍物之间跳转,NPC 只需"被识别为何种类别"(友方/Neutral/敌方),无需传达完整情绪。

#### 障碍物生物 — 辨识特征

| 障碍物 | 主导轮廓 | 警告色比例 | "可读距离" |
|--------|----------|------------|------------|
| **高血糖漩涡** | 螺旋形 | 红色 > 50% | 远景可见 |
| **低血糖冰刺** | 三角锯齿 | 蓝白 < 30% | 中景触发 |
| **炸糖块** | 堆叠立方体 | 琥珀色点缀 | 近景躲避 |
| **血糖 BOSS** | 不规则 + 锯齿王冠 | 深红占主导 | 远景警示 |

**关键规则**:障碍物在剪影层面必须比友方 NPC **更复杂/更尖角**。
- 圆润 = 安全信号;棱角 = 威胁信号(人类进化学本能)
- 这条规则与 Section 2 的"警戒漫画化"原则互补:漫画分镜框是叠加在形状警告之上的二级提示,不是形状本身的替代品。

#### 圆润 vs 棱角 主导原则

**结论:圆润主导,棱角作为警告信号嵌入。**

- 游戏主体是"游乐场",圆润传达无威胁感,降低健康教育内容的心理门槛
- 棱角仅出现在:**障碍物轮廓边缘、Boss 体型、警戒 UI 边框、危险道具(注射器针头)**
- 圆润/棱角比例建议:**85% 圆润资产 / 15% 棱角强调**

### 子主题 2: 环境几何 (Environment Geometry)

**核心主张**:**每关主导几何形态是该关情绪的视觉转译。L1 圆润引导 → L2 棱角刺激 → L3 不规则冰晶压抑,Parallax 三层复杂度递增作为纵深感编码。**

#### 横版关卡主导几何形态

| 关卡 | 主导形状 | 情绪语义 | 形状来源隐喻 |
|------|----------|----------|--------------|
| **L1 医院雪屋** | 圆角丘陵 + 蓬蓬云 | 平静/引导 | 儿童绘本、棉花糖 |
| **L2 高血糖狂暴** | 锐角山脊 + 锯齿云 | 刺激/混乱 | 漫画分镜、嘉年华 |
| **L3 低血糖冰窟** | 不规则冰晶 + 锥刺 | 压抑/黏稠 | 冰川特写、闹钟 |
| **减肥营地** | 几何错落方块 + 圆环 | 清新/欢快 | 积木、健身弹力带 |

**为什么不用纯曲线或纯几何?**
- 纯曲线 (Alto 式) = 无性格,太文艺,不服务"幽默"支柱
- 纯几何 (像素风) = 无亲近感,与 Q 版圆润角色割裂
- **混搭策略**:背景主导形状定义关卡气质,前景角色保持圆润,层次分明

#### Parallax 三层几何复杂度规则

```
Layer 3 (远景) ──── 几何复杂度: 低 (简化形状, 纯色或微弱渐变)
Layer 2 (中景) ──── 几何复杂度: 中 (细节出现, 2-3 色)
Layer 1 (近景) ──── 几何复杂度: 高 (全量细节, 多色, 动效丰富)
```

**实践示例(以 L1 为准)**:
- **远景山峦**:单色圆角三角形序列 + 淡蓝渐变,无纹理
- **中景松树**:几何化三角形树冠 + 圆角矩形树干,2 色(深绿+白边)
- **近景雪坡**:手绘曲线 + 雪纹理 + 雪沫粒子,远景复杂度的 3 倍

**为什么由远及近递增?**
- 人眼近处捕捉细节,远处感知形状
- 近景复杂度高 = 玩家躲避障碍时有足够视觉信息
- 远景简化 = 场景统一感,不抢夺主角注意力

#### 道具/障碍物形态契约

| 类别 | 主导形状 | 边角处理 | 典型色块比 |
|------|----------|----------|------------|
| **收集物** (血糖药丸/能量星) | 圆形/椭圆 + 圆润边框 | 全圆角,无尖角 | 暖色 > 70% |
| **引导道具** (检查点旗) | 三角形帆 + 圆角杆 | 圆润杆,尖角帆 | 中性色 + 亮点 |
| **障碍物** (冰刺/雪块) | 三角形/不规则多边形 | 锐角边缘,外放 | 冷色 + 暗边 |
| **危险警告** (注射器) | 圆润身 + 尖锐针头 | 身圆角,针头锐利 | 红白对比 |

**核心规则**:收集物和障碍物的形状差异度必须 > 60%。

### 子主题 3: UI 形状语法 (UI Shape Grammar)

**核心主张**:**UI 是游戏世界的延伸,而非独立系统。HUD 形状呼应关卡几何,血糖条以"弧形弹药计"为设计锚点,在危急感和卡通感之间取交集。**

**决策:HUD 语言部分呼应游戏世界,但 UI 专用元素独立识别。**

- **呼应游戏世界**:关卡地形轮廓引导 UI 边框(L1 圆弧 → L2 锯齿 → L3 冰晶裂隙)
- **UI 独立元素**:按钮/血槽/计分板保持跨关卡一致性

#### 主 UI 形状选择

| UI 元素 | 推荐形状 | 形状原理 |
|--------|----------|----------|
| **血槽 (Blood Sugar Bar)** | 胶囊形横条 + 端点圆角 | 横向阅读符合横版游戏视线流;胶囊形传递"保护感"而非"威胁感" |
| **暂停按钮** | 圆角矩形 + 双竖线 | 圆角矩形是"按钮"的全球认知原型 |
| **菜单面板** | 圆角矩形气泡 | 气泡形减少屏幕占用,圆角符合圆润总原则 |
| **道具图标边框** | 圆角方形 | 方形边框在道具栏对齐时无缝隙 |
| **分数显示** | 无边框数字 + 投影 | 数字本身是 UI 最简形式 |

#### 血糖条形状:"弧形弹药计"

血糖条采用**弧形弹药计 (Arc Ammo Meter)** 样式:
- **弧形 > 直线横条**:弧形在横版 UI 中更节省中轴空间,留出赛道视野
- **端点圆形锚点**:糖糖的圆润 DNA 延伸到 UI,"满"端是糖糖头像,"低"端是药丸图标
- **填充色**:低血糖 = 深蓝渐变 → 正常 = 翠绿渐变 → 高血糖 = 红紫渐变(Section 2 三色原则的 UI 投影)

**卡通感 + 危急感的平衡点**:
- 危急时血槽边框闪烁 + 弧形压缩动画(漫画弹性)
- 形状本身是圆润的,不传递"医疗仪器"的冰冷感
- 警戒值时出现 ❗❗❗ 漫画符号叠加,不改变形状本身

#### UI 形状一致性规则

1. **所有按钮**:圆角(border-radius ≥ 8px),避免锐角
2. **所有图标边框**:统一 2px 描边,圆角优先
3. **同族元素**:血槽/能量条/进度条共享"胶囊形 + 弧形端点"语法
4. **紧急状态**:形状本身不变,通过颜色/动效/叠加符号传达
5. **字体**:所有 UI 数字/文字使用圆润无衬线体

### 子主题 4: 主形 vs 辅形 (Hero Shapes vs Supporting Shapes)

**核心主张**:**主形是注意力的指挥棒。收集物/紧急提示/终点使用"吸引形状",背景/装饰使用"退隐形状",比例失衡会导致视觉焦距错乱。**

#### 吸引形状 (Attention-Grabbing Shapes)

**形状特性**:高对比色块、暖色、动效、轮廓清晰、几何单一

| 主形目标 | 推荐形状 | 物理属性 | 设计原因 |
|---------|----------|----------|----------|
| **收集物** | 圆形/椭圆 + 高饱和暖色 | 静止或轻微上下浮动 | 圆润 = 可亲,暖色 = 奖励暗示 |
| **紧急提示** | 三角形/菱形 + 高对比色 | 闪烁/脉冲动效 | 三角形是"警示原型"(交通标志同理) |
| **终点旗** | 三角形 + 明亮色 | 静止 | 几何单纯,远处即可辨认 |
| **Boss 出现** | 不规则多边形 + 描边 | 放大出现动画 | 不规则 = 威胁感,描边切割聚焦视线 |

#### 退隐形状 (Background-Receding Shapes)

**形状特性**:低饱和、冷色、简化轮廓、与背景融合

| 辅形目标 | 推荐形状 | 设计原因 |
|---------|----------|----------|
| **Parallax 远景** | 简化三角形/圆弧,无描边 | 无轮廓线,自然融入天空 |
| **装饰性云朵/雪沫** | 蓬蓬云 (puffy cloud) — 多个圆相切 | "棉花糖形" = 低威胁视觉原型 |
| **次要地形** | 圆润山丘,无尖锐边缘 | "安全地面"隐喻,与近景障碍物形成对比 |
| **雪地脚印/痕迹** | 简化椭圆/胶囊,无边界 | 环境叙事,不需成为注意焦点 |

#### 主形应用规则 (5 条)

1. **主形面积占比 ≤ 15% 屏幕** — 超标则缩小图标或减少数量
2. **主形必须有运动属性** — 静态主形容易被忽略,加浮动(3-5px, 0.8s)/脉冲/摇摆
3. **辅形不得使用暖色高饱和** — 防止"前景污染"
4. **主形与辅形的形状差异度 ≥ 40%** — 防止视觉误判
5. **警戒状态时临时提升辅形为主形** — 形状不变,注意力权重动态重分配

### 形状决策测试 (示例)

**场景**:团队设计师提出将 L2 的"高血糖暴风雪"中的雪花设计为**尖锐六芒星形**,理由是"雪花就是六芒星,这是知识正确"。

**应用 Section 1 原则裁决**:
> "卡通奇幻游乐场 — 不让医学写实侵入"

- **裁决**:否决
- **理由**:六芒星形是"写实雪花"的医学/物理正确,但在 Section 2 的 L2 "狂热/混乱"情绪下,尖锐形状会强化恐慌感而非"卡通刺激感"
- **替代方案**:保留**六芒星结构**(知识性),但将尖端改为**圆润化**(卡通化) —— 仍然是"雪花的认知符号",但形状语言服务于"狂热但可控"的情绪目标

**形状测试的日常化**:每引入一个资产,需由设计师自问"这个形状在 Section 1 三原则下是否通得过?"通不过则回炉修改,不依赖主观喜好。

### 子主题 2: 语义颜色词汇 (Semantic Color Vocabulary)

#### 红色家族 — "危机但不绝望"

**核心含义**:高血糖状态、危险但卡通化的敌人、需要避开的障碍。

**为什么不是纯"危险"**:SugarRush 是幽默风格的游戏,完全写实的红色警报会造成恐慌而非参与感。红色在游戏中是"有趣的危机"—— 玩家知道要躲避,但不会感到威胁。医学背景是:血糖过高时视野会模糊、心跳加速,游戏用红色映射这种生理反应,但通过饱和度和卡通描边控制在"刺激但可控"的范围内。

**使用边界**:红色家族绝不用于严肃的医疗场景(如真实的低血糖急救)。它是游戏世界的语言,不是医学替代品。

| 色阶 | 语义 | 游戏内示例 |
|------|------|-----------|
| `#E63946` 鲜红 | 急性危险 / Boss 战 / 超限警报 | L2 跑道主色、boss 糖化身核心 |
| `#FF6B6B` 浅红 | 警戒预告 / 即将到来的障碍 | 障碍物描边渐变、血糖临界闪烁 |
| `#C1121F` 深红 | 危机峰值 / 屏幕震动同步 | L2 高潮段落环境光、谢幕失败渐变起点 |
| `#FF9F1C` 红橙 | 高糖糖晶核心 / 能量补充 | 高糖 collectible 核心、能量道具 |

#### 蓝色家族 — "冷静与干预"

**核心含义**:胰岛素干预、冷静状态、引导与帮助、可逆的错误。

**为什么蓝色=冷静**:医学上胰岛素有"冷却"血糖过高的作用;视觉上蓝绿色调天然降低心率。游戏利用这个对应关系:当玩家看到蓝色元素时,大脑自动关联"安全/可纠正"。L1 医院雪屋用蓝绿白营造消毒室般的干净感,但通过圆润造型和暖黄色高光让它"有趣"而非"医疗恐惧"。

| 色阶 | 语义 | 游戏内示例 |
|------|------|-----------|

### 子主题 4: UI 调色板 (UI Palette)

#### 为什么 UI 颜色与世界颜色不同

游戏世界的颜色服务于"沉浸感"和"情绪引导",UI 颜色服务于"信息可读性"和"操作反馈"。两者可能冲突:

- **雪地背景 + 白色文字**:世界中唐白 (`#F5F0E8`) 是优雅的雪地基底,但作为 UI 文字色会导致可读性灾难。
- **高饱和度世界 + 低饱和度 UI**:L2 的血糖红 (`#E63946`) 是情绪引擎,但用于 UI 背景会干扰玩家读取数字。

**核心原则**:UI 颜色优先可读性,世界颜色优先情绪。两者在边界处(如道具图标)通过描边/光晕分离。

| 元素 | 最低对比度 | 测量方式 |
|------|-----------|--------|
| 正文文字 | 4.5:1 | 文字色 vs 背景色 |
| 大文字(>24px) | 3:1 | 文字色 vs 背景色 |
| UI 图标 | 3:1 | 图标填充色 vs 背景 |
| 血糖数字 | 4.5:1 | 数字色 vs 状态背景 |
| 按钮文字 | 4.5:1 | 文字色 vs 按钮背景 |
| 世界元素叠加 UI | 5:1 | UI 元素 vs 世界背景叠加层 |

**实现建议**:
- 所有 UI 文本不要直接放在游戏世界上,使用圆角卡片背景 (`#2D2A32` 可可黑,80% 透明度) 确保可读性
- 血糖数字使用黑色 (`#2D2A32`) 而非白色,在黄色/绿色背景上对比度更佳
- 警戒状态数字使用深色 (`#2D2A32`),在红色背景上可读

### 子主题 5: 色盲安全 (Colorblind Safety)

#### 为什么色盲安全在 SugarRush 中特别重要

糖尿病教育内容涉及血糖状态判断,颜色的误读可能导致玩家误解游戏传达的健康信息。色盲安全不是"可选适配",而是**教育严肃性**的基本要求。

#### 需要 shape/icon backup 的颜色对

以下颜色对在常见色盲类型中难以区分,**必须**配合形状或图标冗余:

| 颜色对 | 色盲类型 | 游戏中因此可能混淆的元素 | 必须的 shape backup |
|--------|---------|--------------------------|---------------------|
| 红 (`#E63946`) / 橙 (`#FF9F1C`) | Deuteranopia(绿盲,最常见,约 6% 男性) | 高糖糖晶 vs 普通糖晶 | 形状不同:高糖=六边形糖晶,普通=圆形水果 |
| 绿 (`#06D6A0`) / 黄 (`#FFD166`) | Protanopia(红盲,约 2% 男性) | 血糖正常 vs 警戒状态 | 状态图标:正常=✓对勾,警戒=⚠三角形 |
| 蓝 (`#4CC9F0`) / 紫 (`#7B2CBF`) | Tritanopia(蓝盲,极罕见) | 胰岛素道具 vs Boss 能量 | 形状+图标:胰岛素=圆形+注射器图标,boss=不规则+骷髅图标 |

#### 信号必须有非颜色冗余的情况

以下游戏信号**仅靠颜色传达是不够的**,必须叠加声音、动效或形状:

| 信号类型 | 颜色冗余 | 必须叠加的非颜色信号 |
|----------|----------|----------------------|
| 血糖超限警报 | 屏幕红色脉冲 | 低频脉冲声音(60Hz)+ 屏幕震动 |
| 收集到水果 | 绿色 pop-up | "叮"高音效 + 数字增大动效 |
| 胰岛素生效 | 蓝色光芒扩散 | "嘶"中频音效 + 减速慢动作 |

#### 三类色盲快速检查清单

##### Deuteranopia(绿盲,deuteranomaly 为较轻变体)

**原理**:红-绿锥细胞感光色素异常,绿色感受能力降低。是最常见的色盲类型(约 6% 男性,0.4% 女性)。

**自检 HEX 配对**:
```
#E63946 (血糖红) vs #06D6A0 (健康绿) → 在 deuteranopia 视野中几乎相同
#FF9F1C (红橙) vs #FFD166 (阳光黄) → 在 deuteranopia 视野中难以区分
```

**游戏内风险元素**:
- [ ] L2 高糖糖晶(红橙)与普通水果 collectible(绿)在色盲模式下是否可辨?
- [ ] 警戒状态(黄)与危机状态(红)在色盲模式下是否可辨?
- [ ] UI 血糖数字(绿=正常,红=危机)在色盲模式下是否可辨?

**修复方案**:以上所有元素必须使用形状冗余(高糖糖晶=六边形,水果=圆形,警戒=三角形图标,危机=八边形图标)。

##### Protanopia(红盲,protanomaly 为较轻变体)

**原理**:红-绿锥细胞感光色素异常,红色感受能力降低。影响约 2% 男性,0.01% 女性。

**自检 HEX 配对**:
```
#E63946 (血糖红) vs #06D6A0 (健康绿) → 在 protanopia 视野中几乎相同
#E63946 (血糖红) vs #FFD166 (阳光黄) → 在 protanopia 视野中难以区分
```

**游戏内风险元素**:
- [ ] L2 障碍物(红)与 collectible(绿)在色盲模式下是否可辨?
- [ ] 高糖雪花道具(红)与普通 collectible 在色盲模式下是否可辨?
- [ ] Boss 战环境(红紫渐变)在色盲模式下是否仍可辨?

**修复方案**:障碍物使用三角形+暗边描边,collectible 使用圆形+光晕,与颜色无关。

##### Tritanopia(蓝盲)

**原理**:蓝-黄锥细胞感光色素异常,蓝色感受能力降低。极为罕见(约 0.01% 人口),多为先天性。

**自检 HEX 配对**:
```
#4CC9F0 (胰岛素蓝) vs #7B2CBF (葡萄紫) → 在 tritanopia 视野中可能混淆
#48CAE4 (浅蓝) vs #023E8A (深蓝) → 在 tritanopia 视野中对比度降低
```

**游戏内风险元素**:
- [ ] L1 胰岛素道具(蓝)与 L3 冰窟背景(深蓝)在色盲模式下是否可辨?
- [ ] UI 胰岛素图标(蓝)与 UI 危机图标(紫)在色盲模式下是否可辨?

**修复方案**:使用形状冗余(胰岛素=圆形+注射器图标,危机=脉冲+感叹号图标)。
